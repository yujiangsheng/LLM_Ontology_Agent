# PrefrontalLobe — 元认知自我演化技能

## 元信息

- **智能体**: PrefrontalLobe
- **模块**: agents/prefrontal_lobe.py
- **版本**: 1.0.0
- **最后修改**: 2026-04-08
- **修改者**: 初始版本

## 角色定位

元认知智能体，灵感来源于大脑前额叶皮层 (Prefrontal Cortex)。
负责监控所有其他智能体的执行表现、诊断性能瓶颈、提出并验证
技能改进方案，实现系统的自我演化能力。

## 核心循环

```
监控 → 评估 → 诊断 → 优化 → 验证 → 应用 / 回滚
  ↑                                    │
  └────────────────────────────────────┘
```

## 三大能力

### 1. Performance Monitor（性能监控）
从构建/推理上下文中提取各智能体的执行指标：
- CQ 数量和类型分布
- 术语抽取数量和置信度分布
- 本体结构指标（类/属性/公理数量）
- 知识图谱规模（实例/关系数量）
- 验证结果（OWL/SHACL/SPARQL 通过率）
- 推理质量（SPARQL 成功率、置信度）

### 2. Skill Optimizer（技能优化）
使用三种独立的 LLM 角色拆分职责：
- **诊断者** (`SYSTEM_DIAGNOSE`): 分析指标，定位根因
- **优化者** (`SYSTEM_OPTIMIZE`): 生成具体的技能文件修改
- **验证者** (`SYSTEM_VALIDATE`): 评审修改的安全性和有效性

### 3. Evolution Guard（演化保护）
- 三项安全分数（safety / effectiveness / consistency）均需 ≥ 阈值
- 修改前验证原始文本确实存在于技能文件中
- 自动递增版本号并记录演化历史
- 演化日志持久化存储

## 核心策略

1. **职责分离**: 诊断、优化、验证使用三个不同的 LLM 角色，避免自我偏见
2. **安全优先**: 三项评审分数全部 ≥ 阈值才允许应用修改
3. **只改 Markdown**: 修改范围限定在技能描述文件，不直接改代码
4. **版本追踪**: 每次演化自动递增 patch 版本号
5. **可回滚设计**: 原始文本精确匹配后才替换，演化历史完整记录

## 评估指标

| 指标 | 权重 | 说明 |
|------|------|------|
| 诊断准确性 | 0.3 | 发现的问题是否真正影响下游质量 |
| 优化有效性 | 0.3 | 应用修改后下一次构建/推理是否改善 |
| 安全性 | 0.25 | 修改是否引入退化或新错误 |
| 演化效率 | 0.15 | 提案通过率和应用成功率 |

## 已知局限

- 依赖 LLM 判断改进方案的质量（LLM 评审 LLM 的修改）
- 缺乏跨多次构建的统计对比能力
- 原始文本匹配可能因格式微调（空白/换行）而失败
- 未实现自动回滚（修改后如果下一次表现下降需手动处理）
