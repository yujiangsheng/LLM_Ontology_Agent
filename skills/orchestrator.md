# Orchestrator — 多智能体协调技能

## 元信息

- **智能体**: Orchestrator
- **模块**: agents/orchestrator.py
- **版本**: 1.0.0
- **最后修改**: 2026-04-08
- **修改者**: 初始版本

## 角色定位

多智能体协调器，编排 7 个专业智能体完成两大核心流程（本体构建 & 推理），
管理全局上下文、记忆生命周期和产物保存。

## 协调流程

### 流程一: 构建 (build_ontology)
```
领域文档 ─→ CQAgent ─→ TermExtractorAgent ─→ OntologyBuilderAgent
                                                      │
                                                      ▼
                                            KnowledgePopulatorAgent
                                                      │
                                                      ▼
                                               ValidatorAgent
```

### 流程二: 推理 (reason)
```
用户问题 ─→ ReasoningAgent ─→ ExplanationAgent ─→ Markdown 报告
```

## 核心策略

1. **五步管线构建**: CQ → 术语 → OWL 本体 → 知识图谱 → 三类验证
2. **两步推理**: 硬+软混合推理 → 自然语言解释
3. **记忆生命周期**: start_session → 各步记录 → 持久化 → end_session
4. **上下文传递**: 各 Agent 产物存入 self.context，逐步累积
5. **产物保存**: JSON/OWL/TTL 文件保存至 output/ 目录
6. **增量加载**: load_from_output() 支持从已有产物继续推理

## 记忆集成

- 构建开始时调用 `memory.start_session()`
- 每个步骤结果写入工作记忆
- 最终本体加载到持久记忆
- 构建经验和验证知识持久化
- 推理结论写入长期记忆
- 会话结束时摘要归档

## 评估指标

| 指标 | 权重 | 说明 |
|------|------|------|
| 构建成功率 | 0.3 | 五步管线全部完成的比例 |
| 端到端正确性 | 0.3 | 构建+推理全链路的正确率 |
| 记忆利用率 | 0.2 | 记忆系统被有效读写的程度 |
| 产物完整性 | 0.2 | 所有预期产物文件是否都生成 |

## 已知局限

- 单一线性管线，不支持并行构建
- 缺乏步骤间的反馈循环（验证失败不会触发重建）
- 记忆系统目前没有自动淘汰机制
